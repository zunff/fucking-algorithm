## 一、入门级：明显局部最优

这些题适合建立贪心感觉。

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 455 | 分发饼干 | Easy | 排序 + 双指针贪心 |
| 860 | 柠檬水找零 | Easy | 模拟 + 优先保留大钞 |
| 1005 | K 次取反后最大化的数组和 | Easy | 排序 + 优先处理负数 |
| 121 | 买卖股票的最佳时机 | Easy | 维护历史最小值 |
| 122 | 买卖股票的最佳时机 II | Medium | 累加所有正收益 |

### LeetCode 455：分发饼干

**模型**：排序 + 小饼干满足小胃口。

**核心贪心**：能用小饼干满足，就不用大饼干。

适合练习：排序后，从小到大匹配。

### LeetCode 860：柠檬水找零

**模型**：局部找零策略。

**核心贪心**：遇到 20 元时，优先用 10 + 5 找零，而不是 5 + 5 + 5。因为 5 元更万能，要尽量保留。

### LeetCode 122：买卖股票的最佳时机 II

**模型**：拆分收益。

**核心贪心**：只要今天比昨天贵，就赚这一段差价。

```
例如 [1, 3, 5]：
1 -> 3 赚 2
3 -> 5 赚 2
总共赚 4，等价于 1 -> 5 赚 4
```

## 二、区间贪心：高频重点

区间问题是贪心最经典的一类。

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 435 | 无重叠区间 | Medium | 按右端点排序 |
| 452 | 用最少数量的箭引爆气球 | Medium | 按右端点排序 |
| 56 | 合并区间 | Medium | 按左端点排序 |
| 763 | 划分字母区间 | Medium | 维护最远边界 |
| 986 | 区间列表的交集 | Medium | 双指针区间 |

### LeetCode 435：无重叠区间

非常经典。

**核心贪心**：每次选择结束最早的区间。

**为什么？** 因为结束越早，留给后面的空间越大。

```java
Arrays.sort(intervals, (a, b) -> a[1] - b[1]);

int count = 1;
int end = intervals[0][1];

for (int i = 1; i < intervals.length; i++) {
    if (intervals[i][0] >= end) {
        count++;
        end = intervals[i][1];
    }
}
```

这类题的关键词：最多选多少个不重叠区间、最少删除多少个区间、会议安排、活动安排。

### LeetCode 452：用最少数量的箭引爆气球

和 435 很像。

**核心贪心**：按右端点排序，一箭射在当前气球的右边界。只要后面的气球左边界小于等于当前箭的位置，就可以一起射爆。

这个题和 435 可以一起刷。

### LeetCode 763：划分字母区间

**模型**：维护当前片段最远边界。

**核心贪心**：当前片段必须覆盖里面所有字符的最后一次出现位置。

```
right = 当前片段中所有字符最后出现位置的最大值
当 i == right 时，说明这一段可以切开。
```

## 三、双向贪心：左右约束

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 135 | 分发糖果 | Hard | 左右两次扫描 |
| 42 | 接雨水 | Hard | 双指针 / 左右最大值 |
| 11 | 盛最多水的容器 | Medium | 双指针移动短板 |

### LeetCode 135：分发糖果

**模型**：左边约束 + 右边约束

```
左到右一遍 → 右到左一遍 → 取 max
```

这题非常值得反复理解。

### LeetCode 11：盛最多水的容器

双指针贪心经典题。

**核心贪心**：每次移动较短的那根柱子。因为水量由短板决定。如果移动长板，高度不会变高，宽度还变小，结果不可能更优。

### LeetCode 42：接雨水

这个题不一定只算贪心，也可以看成双指针。

**核心思想**：当前位置能接多少水，取决于左边最大高度和右边最大高度的较小值。

常见做法：
- 预处理 `leftMax` 和 `rightMax`
- 双指针 + 动态维护左右最大值

## 四、维护最优边界

这类题不是排序，而是一边遍历，一边维护某个「当前最优值」。

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 55 | 跳跃游戏 | Medium | 维护最远可达位置 |
| 45 | 跳跃游戏 II | Medium | 维护当前层边界 |
| 134 | 加油站 | Medium | 当前油量小于 0 就换起点 |
| 376 | 摆动序列 | Medium | 维护上升/下降趋势 |
| 738 | 单调递增的数字 | Medium | 从后往前修正 |

### LeetCode 55：跳跃游戏

**核心贪心**：维护当前能到达的最远位置 `farthest`。

```
如果 i > farthest，说明当前位置到不了，直接失败。
否则更新：farthest = max(farthest, i + nums[i])
```

### LeetCode 45：跳跃游戏 II

比 55 难一点。

**核心贪心**：每一跳尽量扩展到更远的位置。

常见变量：
- `curEnd`：当前这一跳能覆盖的最远边界
- `farthest`：下一跳能覆盖的最远边界
- `steps`：跳跃次数

当遍历到 `curEnd` 时，说明必须再跳一次。

### LeetCode 134：加油站

经典贪心。

**核心结论**：如果从 `start` 出发，到 `i` 的过程中油量变成负数，那么 `start` 到 `i` 之间的任何点都不可能作为起点。所以直接 `start = i + 1`。

## 五、差值贪心

这类题经常出现「两种选择」，需要按收益差排序。

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 1029 | 两地调度 | Medium | 按 A/B 成本差排序 |
| 406 | 根据身高重建队列 | Medium | 排序 + 插入 |
| 621 | 任务调度器 | Medium | 按频率安排 |
| 767 | 重构字符串 | Medium | 优先安排高频字符 |

### LeetCode 1029：两地调度

**核心贪心**：先假设所有人都去 A，然后选出 N 个人改去 B。

谁最应该去 B？看差值 `costB - costA`，差值越小，说明去 B 越划算。所以按 `costA - costB` 或 `costB - costA` 排序即可。

### LeetCode 406：根据身高重建队列

**核心贪心**：先安排高个子，再安排矮个子。

排序规则：
1. 身高从高到低
2. `k` 从小到大

然后按照 `k` 插入到列表中。

**为什么？** 因为矮个子不会影响高个子的 `k` 值。

## 六、堆贪心：动态选择最大/最小

如果题目要求你「动态选择当前最大/最小」，通常会用堆。

| 题号 | 题名 | 难度 | 模型 |
|------|------|------|------|
| 502 | IPO | Hard | 小根堆 + 大根堆 |
| 871 | 最低加油次数 | Hard | 大根堆 |
| 630 | 课程表 III | Hard | 排序 + 大根堆反悔 |
| 1642 | 可以到达的最远建筑 | Medium | 小根堆分配梯子 |
| 253 | 会议室 II | Medium | 小根堆维护结束时间 |

### LeetCode 253：会议室 II

**核心贪心**：按会议开始时间排序，用小根堆维护当前会议的结束时间。

- 如果当前会议开始时间 >= 堆顶结束时间：可以复用会议室
- 否则：需要新会议室

### LeetCode 630：课程表 III

非常经典的「反悔贪心」。

**核心贪心**：按截止时间排序，先尝试选课。如果总时间超过截止时间，就删掉当前已选课程中耗时最长的一门。

**为什么删最长的？** 因为删掉它能释放最多时间，最有利于后面课程。用大根堆维护已选课程的耗时。

### LeetCode 871：最低加油次数

**核心贪心**：能不加油就不加油。当油不够时，从之前经过的加油站里选择油量最大的加。

这就是典型的：先走，走不动了再反悔选择之前最优资源。用大根堆。

### LeetCode 402：移掉 K 位数字

**核心贪心**：为了让数字最小，应该尽量让高位变小。所以如果当前数字比栈顶小，并且还有删除次数，就弹出栈顶。

模板味道：维护一个单调递增栈。

### LeetCode 316：去除重复字母

和 402 很像。

**核心贪心**：尽量让字典序小的字符靠前。但还要保证每个字符都出现一次。

所以当栈顶字符比当前字符大，且栈顶后面还会出现，就可以弹掉。

## 十、刷的时候重点看什么？

每道贪心题都问自己 4 个问题：

1. **局部最优是什么？**
2. **为什么这个选择不会影响全局最优？**
3. **如果影响了，后面能不能修正？**
4. **这题属于哪个贪心模型？**

```
435 无重叠区间
  局部最优：选结束最早的区间
  原因：结束越早，留给后面的空间越大
  模型：区间右端点排序

135 分发糖果
  局部最优：只给刚好满足相邻关系的最少糖
  原因：每个方向的约束可以独立满足
  模型：双向扫描

630 课程表 III
  局部最优：超时后删掉耗时最长的课
  原因：释放最多时间，最不影响课程数量
  模型：反悔贪心 + 堆
```
