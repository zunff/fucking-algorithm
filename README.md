## 动态规划
### 两 字符串/数组 的最大公共子序列长度
子序列不要求元素是连续的，例如 ace 是 abcde 的子序列，以下以数组为例
1. 状态定义：dp[i][j] 为nums1的前i个元素 和 nums2的前j个元素的 最大子序列长度
2. 状态转移方程：
   ```text
   {
      当 nums1[i] == nums2[j] 时：dp[i][j] = dp[i - 1][j - 1] + 1
      当 nums1[i] != nums2[j] 时：dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
   }
    ```
3. 答案：dp[n][m]


### 两 字符串/数组 的最大子数组/子串长度
子数组/子串要求元素是连续的，例如 abc 是 abcde 的子数组，以下以数组为例
1. 状态定义：dp[i][j] 为nums1的以元素i结尾 和 nums2以元素j结尾的 最长子数组长度
2. 状态转移方程：
   ```text
   {
      当 nums1[i] == nums2[j] 时：dp[i][j] = dp[i - 1][j - 1] + 1
      当 nums1[i] != nums2[j] 时：dp[i][j] = 0
   }
    ```
3. 答案：max(dp[i][j])
4. 原因：方便下一个状态拼接时，如果nums1[i-1] nums2[j-1]不相同，直接将其视为0，因为不连续了

### dp数组的定义
1. 不要习惯性dp[i][j] = new int[n+1][m+1]，具体要看状态转移方程，如果状态转移方程依赖 i - 1 和 j - 1 才这样定义
2. 如果状态转移方程依赖 i + 1 和 j - 1 这类，更推荐 dp[i][j] = new int[n + 2][m + 2]

### dp 数组的遍历顺序
1. 也要根据状态转移方程来，如果 状态转移依赖 i - 1，就 i++，如果 状态转移依赖 i + 1 就 i--
2. j 同理

## 双指针 / 滑动窗口（Two Pointers / Sliding Window）
> 适用前提：处理的是连续区间（子数组/子串/区间），并且约束/目标能在指针移动时增量维护。

### 1. 什么时候用双指针
1. **连续性问题**：题目问“某个子数组/子串/区间”的最短、最长、个数。
2. **窗口内信息可维护**：用计数表/哈希表/和/不同元素数等，能在 r++ 或 l++ 时O(1) 更新。
3. **存在单调性/可收缩性**（常见） 
   - “当窗口满足条件时，可以尝试缩小左边界，并且不会漏解”
   = 典型：最小覆盖、最多K种、最多K个不同、无重复、和/乘积不超过K（通常要求非负/正数保证单调）
### 2. 两类经典模板
1. **可变长滑窗：满足条件就收缩（最短/最优）**
   - 目标：最短满足 / 在满足条件下尽量缩
   - 结构：
   ```test
   l = 0
   for r in [0..n-1]:
       把 s[r] 加入窗口并更新窗口状态
       while 窗口满足条件:
           更新答案（通常取更短）
           把 s[l] 移出窗口并更新窗口状态
           l++
   ```
   - 例子：最小覆盖子串、和 ≥ K 的最短子数组（非负时）、满足包含关系的最短区间
2. **可变长滑窗：不满足就收缩（最长/最多）**
   - 目标：最长满足 / 保持窗口始终合法
   - 结构：
   ```text
   l = 0
   for r in [0..n-1]:
       把 s[r] 加入窗口并更新窗口状态
       while 窗口不满足条件:
           把 s[l] 移出窗口并更新窗口状态
           l++
       更新答案（通常取更长）
   ```
   - 例子：无重复字符最长子串、最多 K 种字符的最长子串、子数组和 ≤ K 的最长（非负时）
3. **窗口状态怎么设计（常见“可维护量”）**
   - 计数类：cnt[ch] / map[ch]（出现次数、是否超标）
   - 种类数：distinct（窗口中不同元素个数）
   - 满足度：formed/required、或“当前是否超额”
   - 聚合量：窗口和 sum、乘积 prod（注意单调性条件）、最大/最小值（通常需单调队列辅助）
4. **什么时候不该硬用双指针**
   - 没有单调性：例如包含负数时，“窗口和 ≥ K 的最短子数组”普通滑窗不成立（通常要前缀和 + 单调队列/二分等）。
   - 目标不是连续区间：子序列、编辑距离、匹配、切分最优 —— 更像 DP。
   - 约束无法增量维护：判断窗口是否满足条件需要回扫窗口（会退化成O(n^2)），说明窗口状态设计不对或不适合滑窗。
5. **一句话速记**
   - 连续区间 + 能增量维护约束/答案 → 双指针
   - 非连续/多分支决策/需要记忆历史状态 → DP

## 并查集（Union-Find / DSU）
> 解决的问题：动态连通性，在不断“加边/合并集合”的过程中，快速回答：两点是否在同一集合？合并后集合如何变化？

> 典型适用：无向图连通性、无向图判环、最小生成树（Kruskal）、连通块数量、冗余连接（684）。

1. 核心操作与语义 
   1. find(x)：找代表元（根）
      - 每个集合用一棵树表示，根节点是该集合的代表。 
      - find(x) 返回 x 所在集合的根。
   2. union(a, b)：合并两个集合 
      - 把 a 的根和 b 的根连起来（合并两棵树）。
      - 若本来就在同一集合，合并无效。
   3. connected(a, b)：是否连通
      - find(a) == find(b)
2. 两个关键优化
   1. 路径压缩（Path Compression） 
      - 在 find(x) 时，把 x 到根路径上的点直接挂到根上，后续查询更快。

   2. 按秩/按大小合并（Union by Rank/Size）
      - 合并时把“矮树/小集合”挂到“高树/大集合”下面，避免退化成链。

   > 同时使用后，均摊复杂度接近常数： O(α(n))（反阿克曼函数，实际可视为常数）

3. 什么时候用并查集
   1) ✅ 适合
      1. 无向图加边判环
         - 处理边 (u,v)：若 find(u)==find(v)，这条边会形成环。
         - LeetCode 684 冗余连接：标准解。
      2. 连通块数量 / 动态合并
         - 初始 n 个点 n 个集合，每 union 一次可能让连通块数 -1。
      3. Kruskal 最小生成树
         - 按边权排序，若两端不连通就选这条边并 union。
      4. “等价关系”归类
         - 例如字符串/变量等价、账号合并等。
   2) ❌ 不适合
      1. 有向图判环（如 207 课程表）
         - DSU 的“同集合”不等价于“存在有向环”，会误判。
         - 有向图判环：Kahn / DFS 三色。
      2. 需要删除边、回滚历史（除非用可回滚 DSU）
         - 普通 DSU 不支持高效“撤销 union”。

## 二分答案
> 适用场景：题目问法为「最大值最小化」或「最小值最大化」，答案有明确范围且连续，且能快速判断候选值是否可行。

### 1. 核心特征（满足即考虑二分答案）
1. **问法特征**：求「最大值最小」或「最小值最大」
2. **范围特征**：答案有明确的上下界，且在范围内连续
3. **判定特征**：给定任意候选值，能在O(n)或类似低复杂度下判断是否可行
